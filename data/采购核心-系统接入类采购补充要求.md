# 采购核心-系统接入类采购补充要求

## 什么场景属于系统接入类采购
如果供应商需要做以下任一动作，应视为系统接入类采购：

1. 接入公司 CRM、ERP、OA、财务、客服、工单系统。
2. 调用公司 API 或推拉业务数据。
3. 使用公司单点登录、权限系统或身份认证。
4. 接入日志、埋点、告警、监控平台。

## 采购阶段应额外索取的信息
1. 接入方式说明：API、SDK、Webhook、SFTP、数据库同步等。
2. 权限范围：只读、读写、后台管理或运维权限。
3. 接入环境：测试环境、生产环境、灰度环境。
4. 数据范围：会接触哪些表、字段、对象或业务记录。
5. 回传与缓存策略：数据是否落地、保存多久、谁可以访问。

## 为什么它容易和普通SaaS混淆
很多业务方会说“只是装个工具”，但一旦涉及系统接入，风险就不再只是普通软件采购，而是技术与数据边界问题。

## 采购应如何判断
如果供应商需要接入系统，即便预算不高、功能简单，也应优先考虑：
- 是否触发安全复核。
- 是否需要补齐架构和数据流资料。
- 是否需要明确接口权限边界。
