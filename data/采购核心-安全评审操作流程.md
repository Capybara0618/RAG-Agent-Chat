# 采购核心-安全评审操作流程

## 什么时候应该想到安全评审
只要供应商不是单纯线下服务，而是涉及下面这些情况，采购就要先想到安全评审：

1. 要接公司系统、接口、SSO、日志平台或其他技术环境。
2. 会接触客户数据、员工数据、工单数据、订单数据、会话数据等业务数据。
3. 属于在线软件、SaaS、托管平台、云服务、外部开发组件或外部 AI 服务。
4. 业务方希望快速上线，但技术边界和数据边界还没讲清。

## 启动安全评审时先要什么材料
采购启动安全评审时，建议一次性向供应商索取：

1. 安全能力说明或安全问卷。
2. 产品架构、部署方式或系统边界说明。
3. 数据流说明，包括采集什么、传到哪里、谁能访问。
4. 数据存储地点、部署地域、云资源或机房信息。
5. 是否存在分包商、云服务商、第三方处理方。
6. 安全事件通知机制，特别是发现问题后多久通知。

## 安全复核真正关注什么
安全团队通常不会只问“你们安不安全”，而是重点问：

1. 会不会碰到我们的数据，碰到哪类数据。
2. 数据放在哪里，是否涉及跨区域或跨境。
3. 是否接入生产环境，权限边界怎么控制。
4. 有没有第三方或分包处理链路。
5. 如果发生安全事件，对方是否承诺及时通知和配合处置。

## 哪些信号说明风险偏高
下面这些都属于高风险信号：

1. 供应商说会处理数据，但说不清数据流和存储位置。
2. 供应商说会接系统，但不给架构图、接口边界或部署说明。
3. 供应商承认有第三方处理，却不解释谁在处理、边界是什么。
4. 供应商拒绝提供安全问卷，或对事件通知义务表述模糊。

## 什么时候应该暂停推进
出现下列情况时，采购不应继续往前推：

1. 供应商拒绝补安全材料。
2. 供应商无法说明数据流向、部署位置或关键架构信息。
3. 供应商不愿承诺安全事件通知机制。
4. 供应商对分包安排、第三方处理或云资源来源完全说不清。

## 典型场景示例
### 场景一：客服 SaaS
如果一个客服 SaaS 需要对接 CRM、读取客户聊天记录并提供报表，这种情况必须做安全评审，重点关注数据流、部署和事件通知。

### 场景二：接口类工具
如果一个外部工具需要调用公司 API 或接入日志系统，即便合同金额不大，也不能跳过安全评审。

### 场景三：供应商口头承诺“上云部署”
“上云部署”这种说法不够，需要继续追问部署地域、数据驻留位置、云厂商和是否有第三方处理。
