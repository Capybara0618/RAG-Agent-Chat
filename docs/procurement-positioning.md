# 采购版项目定位

## 项目新定位

**ProcureOps Copilot：面向采购合同审查与供应商准入的可审计 RAG + Agent 平台**

## 解决的问题

- 采购、法务、信息安全和业务之间协作链条长，合同审查和供应商准入经常依赖经验。
- 标准模板、红线指引、审批矩阵、安全评审 SOP 分散在多个文档里，难以快速定位。
- 普通聊天机器人无法给出“引用依据 + 风险提示 + 升级建议”，不适合合同审查场景。

## 当前覆盖的采购场景

- 供应商准入与尽职调查
- 合同核心红线条款审查
- 标准模板与供应商回传版本对比
- 安全评审流程与升级条件
- 审批矩阵与法务/安全升级判断

## 面试时怎么讲

这不是一个泛知识问答 demo，而是一个采购协同场景下的制度与条款助手：

1. 能回答供应商准入、法务红线和审批矩阵问题。
2. 能对比标准合同模板与供应商红线版本，找出缺失条款和风险点。
3. 能在证据不足时要求澄清，在涉及最终法务签批时主动拒答。
4. 会保留 trace、引用、失败归因和评测结果，适合审计要求高的企业内部流程。

## 推荐的一句话介绍

我做了一个面向采购合同审查与供应商准入场景的 RAG + Agent 平台，用来回答制度流程问题、对比合同条款差异、识别核心红线条款，并保留完整的引用和 trace，帮助采购、法务和安全团队提高审查效率。
