# 架构说明

## 核心流程

1. 文档先被上传或抓取进系统，并被标准化成结构化 section。
2. section 会继续被切分成带标题和位置信息的 chunk。
3. 每个 chunk 都会生成确定性的 embedding 和关键词索引。
4. Agent 工作流会依次完成意图路由、检索规划、实际检索、答案生成和证据校验。
5. 每次问答都会保存 trace 记录和逐步执行轨迹，方便审计和复盘。

## 面试表达方式

- 这不是一个普通聊天机器人，而是一个面向知识运营场景的平台，覆盖知识接入、检索、工作流编排、可追踪性、反馈闭环和评测。
- 当前本地版本默认使用 SQLite 和确定性 embedding，目的是让演示和学习成本尽量低。
- 后续的生产化升级路径可以是 PostgreSQL + pgvector + Redis + 外部模型 API。
