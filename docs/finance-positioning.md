# 金融版项目定位

## 新项目定位

将项目从通用企业知识助手收束为：

**FinKnowledge Copilot：面向金融机构制度、合规与风控场景的可审计 RAG + Agent 平台**

## 解决的问题

- 金融机构内部制度多、版本更新快，业务人员难以快速定位最新口径。
- KYC、AML、适当性、研报披露、交易监测等场景对“引用依据”和“审计留痕”要求很高。
- 传统聊天机器人容易给出没有依据的回答，不适合高合规场景。

## 当前覆盖的核心金融场景

- 财富管理产品适当性校验
- 客户开户 KYC 与高风险客户增强尽调
- 交易监测与可疑交易上报流程
- 研究报告披露与黑窗期控制

## 面试时怎么讲

可以把项目描述成一个“金融机构内部制度与合规助手”，重点强调：

1. 不是泛问答，而是围绕金融制度和风控流程。
2. 不是只返回答案，而是返回引用、置信度、trace 和审计记录。
3. 不是只做 prompt，而是有检索规划、证据对比、引用校验和评测闭环。
4. 对个股推荐、收益预测等超出制度知识边界的问题会主动拒答。

## 推荐的一句话介绍

我做了一个面向金融机构合规与风控知识场景的 RAG + Agent 平台，用来回答 KYC、AML、适当性、研报披露等问题。系统会先检索制度证据，再生成带引用的回答，并保留 trace 和评测结果，适合高审计要求的金融场景。
